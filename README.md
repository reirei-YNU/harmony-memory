# harmony-memory

ピアノ練習の録音を **曲ごと・レベルごと** に整理できる、個人用の学習記録 PWA（インストール可能なWebアプリ）です。

ログインもアカウントも不要です。すべてのデータはあなたのブラウザ内（IndexedDB）だけに保存され、外部には一切送信されません。

## 主な機能

- **録音**: ブラウザのマイクから直接録音。Opus (48kbps) で圧縮保存するため、たくさんのテイクを保存できます。
- **曲ごとの見出し**: 録音は必ず「曲」に紐づき、曲名を見出しにして一覧表示されます。
- **レベルラベル**: 各録音に「初級 / 中級 / 上級 / 発表会用」などのレベルラベルを付けられます（自由に追加可能）。
- **並び替え・絞り込み**: 「曲ごと」「レベルごと」の表示切り替え、レベルでの絞り込み、新しい順・古い順の並び替えに対応。
- **曲ごとの目標・メモ**: 各曲に自由に目標やメモを書いて残せます。
- **練習カレンダー**: 直近12週間の練習日をヒートマップで可視化します。
- **連続記録**: 何日連続で練習したか（ストリーク）を表示します。
- **練習時間の集計**: 今週・今月の合計練習時間を表示します。
- **ストレージ使用量の可視化**: この端末で使っている保存容量をメーターで確認できます。
- **PWA**: スマホやPCに「アプリ」としてインストールできます。オフラインでも動作します。

## 技術構成

- [Vite](https://vitejs.dev/) + React + TypeScript
- **IndexedDB**（ブラウザ内蔵のデータベース）— サーバー・アカウント一切不要
- [vite-plugin-pwa](https://vite-pwa-org.netlify.app/) — インストール可能なPWA化

外部サービスへの依存が一切ないため、GitHub Pages のような静的ホスティングだけで完結します。

## セットアップ

```bash
npm install
npm run dev
```

`http://localhost:5173/harmony-memory/` を開いて動作を確認できます。

### GitHub Pages へのデプロイ

`main` ブランチに push すると `.github/workflows/static.yml` が自動でビルド・デプロイします。リポジトリの **Settings > Pages** で Source を "GitHub Actions" に設定してください。デプロイが終わると `https://<ユーザー名>.github.io/harmony-memory/` でスマホからもPCからもアクセスできます。

## 使い方

1. トップ画面の「+ 録音する」から曲を選択（または新規追加）し、レベルを選んで録音します。
2. トップ画面で「曲ごと」「レベルごと」の表示を切り替えたり、レベルで絞り込んだりして練習記録を振り返れます。
3. 曲の見出しの下にある「目標・メモを追加...」から、その曲の目標や気づきを書き留められます。
4. カレンダーや連続記録、練習時間の集計で、自分の練習の積み重ねを確認できます。

## 注意事項

- データはこのブラウザ・この端末にしか保存されません。ブラウザのデータを消去したり、別の端末・別のブラウザで開いたりすると録音は見られなくなります。
- 他の人と共有する機能は現在ありません。
